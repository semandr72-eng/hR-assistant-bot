# 🤖 Telegram-бот HR-ассистент с RAG

Мультимодальный AI-ассистент для корпоративной поддержки сотрудников с использованием Retrieval-Augmented Generation (RAG).

![Python](https://img.shields.io/badge/Python-3.10+-blue.svg)
![License](https://img.shields.io/badge/License-MIT-green.svg)
![Telegram Bot API](https://img.shields.io/badge/Telegram-Bot%20API-2CA5E0.svg)

## ✨ Особенности

- **📝 Текстовые ответы** - GPT-4o для общих вопросов
- **🎤 Голосовые сообщения** - STT (Whisper) + TTS для голосового взаимодействия
- **📸 Анализ изображений** - GPT-4 Vision для распознавания объектов и документов
- **📚 База знаний (RAG)** - поиск по корпоративным документам
- **👔 HR-ассистент** - специализированная поддержка для новых сотрудников
- **🎨 Генерация изображений** - DALL-E 3 для создания визуализаций

## 🚀 Быстрый старт

### Требования

- Python 3.10+
- Telegram Bot Token
- OpenAI API Key
- ffmpeg (для работы с аудио)

### Установка

1. **Клонировать репозиторий:**
```bash
git clone https://github.com/yourusername/hr-assistant-bot.git
cd hr-assistant-bot
```

2. **Создать виртуальное окружение:**
```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
venv\Scripts\activate     # Windows
```

3. **Установить зависимости:**
```bash
pip install -r requirements.txt
```

4. **Настроить переменные окружения:**

Создайте файл `.env`:

```env
# Telegram Bot Configuration
TELEGRAM_BOT_TOKEN=your_bot_token_here

# OpenAI Configuration
OPENAI_API_KEY=your_openai_api_key_here

# Optional: Use ProxyAPI for Russia (without VPN)
USE_PROXYAPI=false

# Bot Mode (text, voice, vision, rag, hr)
BOT_MODE=text

# Default Voice (alloy, echo, nova, fable, onyx, shimmer)
DEFAULT_VOICE=alloy

# Logging Configuration
LOG_LEVEL=INFO
```

5. **Подготовить базу знаний:**

Поместите документы в папку `data/documents/`:
- `.pdf` файлы
- `.txt` файлы
- `.md` файлы

6. **Запустить бота:**
```bash
python main.py
```

## 📖 Использование

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и обзор возможностей |
| `/help` | Полная справка по всем командам |
| `/mode <режим>` | Переключение режима работы |
| `/voice <голос>` | Выбор голоса для TTS ответов |
| `/voices` | Список доступных голосов |
| `/reset` | Очистка истории диалога |
| `/stats` | Статистика базы знаний |
| `/hr` | Справка HR-ассистента |
| `/hr/reset` | Сброс контекста HR-диалога |
| `/hr/status` | Статус онбординга |
| `/image <промпт>` | Генерация изображения |

### Режимы работы

Бот поддерживает переключение между режимами через `/mode`:

| Режим | Описание |
|-------|----------|
| `text` | Обычный текстовый диалог (по умолчанию) |
| `voice` | Голосовой ввод + голосовой ответ |
| `vision` | Анализ изображений |
| `rag` | Работа с базой знаний (автоматически включён) |
| `hr` | HR-ассистент для новичков |

### Голоса для TTS

- `alloy` - нейтральный (по умолчанию)
- `echo` - мужской
- `nova` - женский
- `fable` - мужской (британский акцент)
- `onyx` - глубокий мужской
- `shimmer` - женский (тёплый)

## 🔧 RAG Архитектура

### Процесс обработки запроса

```
1. ПОЛЬЗОВАТЕЛЬ → Вопрос: "Сколько отделов в Ашане?"
                    │
                    ▼
2. SEARCH → Векторный поиск в ChromaDB
            - Cosine Similarity
            - Top-K: 3 результата
                    │
                    ▼
3. CONTEXT → Подготовка контекста
             - Объединение чанков
             - Добавление метаданных
                    │
                    ▼
4. GENERATION → GPT-4o генерация ответа
                - Только на основе контекста
                - Без выдумывания
                    │
                    ▼
5. RESPONSE → "В структуре компании Ашан 20 функциональных отделов."
```

### Конфигурация RAG

| Параметр | Значение | Описание |
|----------|----------|----------|
| `RAG_CHUNK_SIZE` | 1000 | Размер чанка в символах |
| `RAG_CHUNK_OVERLAP` | 200 | Перекрытие между чанками |
| `RAG_TOP_K` | 3 | Количество результатов поиска |
| `EMBEDDINGS` | OpenAI | text-embedding-ada-002 |
| `VECTOR_STORE` | ChromaDB | Локальное SQLite хранилище |

## 🛠️ Разработка

### Добавление новых документов в базу знаний

1. Поместите файл в `data/documents/`
2. Поддерживаемые форматы: `.pdf`, `.txt`, `.md`
3. Бот автоматически проиндексирует при запуске
4. Проверьте статус: `/stats`

### Расширение функциональности

1. **Новый обработчик команд:**
   - Создайте файл в `handlers/`
   - Используйте декораторы `@bot.message_handler`
   - Импортируйте в `main.py`

2. **Новый сервис:**
   - Создайте файл в `services/`
   - Используйте глобальный клиент `openai_client`
   - Добавьте в `router.py`

3. **Новый режим:**
   - Добавьте в `config.py` -> `BotMode`
   - Создайте обработчик в `handlers/`
   - Реализуйте в `router.py`
